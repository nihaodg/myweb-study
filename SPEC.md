# CTF-Web 常见漏洞原理学习平台 - 项目规范

## 1. 概念与愿景

一个交互式网络安全学习平台，通过可视化的方式让学习者理解 Web 漏洞的原理。平台提供漏洞原理解析、多语言代码展示、交互式漏洞复现和修复实验，帮助学习者从理论到实践全面掌握 CTF-Web 技能。

## 2. 设计语言

### 视觉风格
- **参考风格**：你提供的 Tailwind CSS 黑暗主题风格（`bg-gray-900`, `bg-gray-800`, 彩色高亮）
- **主题**：网络安全/黑客风格，终端感

### 色彩系统
```
背景主色:    bg-gray-900 (#111827)
卡片背景:    bg-gray-800 (#1f2937)
边框色:      border-gray-700 (#374151)
主色调:      blue-400/500 (#60a5fa)
危险/漏洞:   red-500/600 (#ef4444)
修复/安全:   green-400/500 (#4ade80)
警告色:      yellow-400/500 (#facc15)
代码背景:    bg-gray-950 (#030712)
```

### 字体
- 代码：`font-mono` (等宽字体)
- 正文：`font-sans` (系统无衬线字体)

### 组件模式
- 卡片式布局，圆角 `rounded-lg`，边框 `border border-gray-700`
- 悬停效果：`hover:bg-gray-700`，过渡 `transition-all`
- 激活状态：`active:scale-95`
- 阴影：`shadow-lg`

## 3. 页面结构

### 3.1 首页 (index.html)
- 顶部标题 + 简介
- 漏洞卡片网格展示
- 每个卡片包含：漏洞名称、难度标签、简介

### 3.2 漏洞详情页
分为三个主要区域：

**左侧面板：**
- 代码展示区（支持 Java/PHP/Python/Go 多语言切换 Tab）
- 漏洞原理说明
- 修复方案说明

**右侧面板：**
- 可视化展示区（根据漏洞类型不同）
- 交互式输入区（用于漏洞复现）
- 状态反馈区（复现结果/修复验证）

**底部：**
- 步骤解析区（类似你提供的步骤说明）

## 4. 漏洞类型（5-8个核心漏洞）

| 漏洞类型 | 目录名 | 可视化重点 |
|---------|--------|-----------|
| SQL注入 | sqli | SQL语句构建、黑白名单对比 |
| XSS | xss | DOM结构、脚本注入点 |
| CSRF | csrf | 请求流程图、Token验证 |
| 文件上传 | file-upload | 文件名解析、白名单过滤 |
| SSRF | ssrf | URL解析、协议限制 |
| 命令注入 | command-injection | 命令拼接、管道符利用 |
| 认证失效 | auth-bypass | 会话管理、权限验证流程 |
| XXE | xxe | XML解析、实体引用 |

## 5. 漏洞详情页结构（统一模板）

每个漏洞详情页包含以下模块：

### 5.1 代码展示区
- 语言切换 Tab（Java / PHP / Python / Go）
- 漏洞代码 vs 修复代码切换
- 代码行号
- 高亮标记漏洞行

### 5.2 漏洞原理区
- 文字说明 + 配图
- 攻击向量示意
- 漏洞成因分析

### 5.3 交互实验区
- 输入框（根据漏洞类型定制）
- "攻击" 按钮
- 结果展示区（成功/失败）
- "修复代码" 按钮
- 修复后重新测试

### 5.4 步骤解析区（类似栈溢出的步骤说明）
- 分步骤展示漏洞原理
- 当前步骤高亮
- 上一步/下一步控制

## 8. 验收标准

### 功能验收
- [ ] 首页展示所有漏洞列表
- [ ] 点击漏洞进入详情页
- [ ] 多语言代码切换正常
- [ ] 漏洞代码与修复代码可切换
- [ ] 交互输入能触发漏洞演示
- [ ] 步骤解析可逐步展示

### 视觉验收
- [ ] 黑暗主题风格统一
- [ ] Tailwind CSS 样式规范
- [ ] 响应式布局（桌面端展示完整，移动端自适应）
- [ ] 动画过渡流畅

### 内容验收
- [ ] 每个漏洞有完整原理说明
- [ ] 代码示例真实可运行（语法正确）
- [ ] 攻击/修复演示符合实际安全场景
